## Dependency Update Review

**Verdict:** Yellow Caution
**Recommendation:** Merge after checks
**Confidence:** Medium

### Executive summary

This PR updates the runtime image for the Dependency-Track custom metrics exporter from `docker.io/python:3.12-slim` to `docker.io/python:3.14-slim` (with digest pin). The main risk is Python-runtime compatibility across a skipped minor line (3.13) for a production monitoring component, even though the exporter script is small and only uses Python stdlib modules. Upstream notes show notable interpreter and language-behavior changes between 3.12 and 3.14, but none directly target the modules used by this exporter. Recommended action is to merge after a focused runtime smoke check in-cluster (exporter starts, `/metrics` serves, and DT alert rules remain green).

### Local impact

The PR changes one file: `kubernetes/apps/security/dependency-track/app/metrics-exporter/deployment.yaml` (`image` field only). This workload runs a custom Python script sourced from `kubernetes/apps/security/dependency-track/app/metrics-exporter/configmap.yaml` and exposes Prometheus metrics through `kubernetes/apps/security/dependency-track/app/metrics-exporter/service.yaml` and `.../servicemonitor.yaml`. The script uses stdlib-only imports and a simple HTTP server loop, so compatibility risk is moderate but bounded. Blast radius is monitoring/security visibility (Dependency-Track metrics and downstream alerts), not direct application data writes.
