# Talos node operations

Nodes run **Talos Linux**, machine configs are generated by **talhelper**. Everything lives in `talos/`:

- `talos/talconfig.yaml` — node definitions (IP, install disk, `talosImageURL`, per-node patches)
- `talos/talenv.yaml` — **source of truth** for `talosVersion` + `kubernetesVersion` (README/techdocs version numbers are stale — don't trust them)
- `talos/patches/` — machine-config patches
- `talos/clusterconfig/` — generated per-node configs + **talosconfig** (`talos/clusterconfig/talosconfig`, used by `talosctl`)
- `talos/talsecret.sops.yaml` — SOPS-encrypted; never edit or decrypt

| soyo-2 | 10.0.0.21 | control-plane | etcd |
| soyo-3 | 10.0.0.22 | control-plane | etcd — **address by IP `10.0.0.22`** (hostname resolution is flaky) |
| fringe-workstation | 10.0.0.23 | worker | write-heavy workloads land here |

**Control-plane nodes soyo-1/2/3 share one disk** for etcd + everything. Schedule write-heavy workloads on the worker **`fringe-workstation`** via hard `requiredDuringScheduling` nodeAffinity on `node-role.kubernetes.io/control-plane DoesNotExist`.

## ⚠️ Applying config always reboots here

Use the **safe** wrapper (drain → apply → wait Ready → uncordon), never the bare apply:

```bash
task talos:apply-node-safe IP=<ip> HOSTNAME=<hostname>
```

`task talos:apply-node IP=<ip>` does **not** drain — avoid it; it reboots with workloads still scheduled. For soyo-3 pass `IP=10.0.0.22`.

## Recipes — `.taskfiles/talos/Taskfile.yaml`

```bash
task talos:generate-config                            # talhelper genconfig (run after editing talconfig/patches/talenv)
task talos:apply-node-safe IP=<ip> HOSTNAME=<name>    # safe rolling machine-config apply (drains first)
task talos:upgrade-node IP=<ip>                       # upgrade Talos on one node (image+version from talconfig/talenv)
task talos:upgrade-k8s                                # upgrade Kubernetes to talenv kubernetesVersion
task talos:reset                                      # DESTRUCTIVE: wipe nodes back to maintenance mode
```

**Version bumps:** edit `talos/talenv.yaml` (`talosVersion` / `kubernetesVersion`), run `task talos:generate-config`, then roll `task talos:upgrade-node IP=<ip>` per node (one at a time) and/or `task talos:upgrade-k8s`.

## Read-only diagnostics

```bash
mise exec -- talosctl --talosconfig talos/clusterconfig/talosconfig -n <ip> health
mise exec -- talosctl --talosconfig talos/clusterconfig/talosconfig -n <ip> dmesg
mise exec -- talosctl --talosconfig talos/clusterconfig/talosconfig -n <ip> get members
```
