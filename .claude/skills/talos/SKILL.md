# Talos node operations

Talos Linux, machine configs generated by **talhelper**. In `talos/`: `talconfig.yaml` (node defs), `talenv.yaml` (**source of truth** for `talosVersion`/`kubernetesVersion` — docs are stale), `patches/`, `clusterconfig/` (generated + `talosconfig` for talosctl), `talsecret.sops.yaml` (never edit/decrypt).

## Nodes
| Node | IP | Role | Hardware | Notes |
|---|---|---|---|---|
| soyo-1 | 10.0.0.20 | control-plane | N150 4C/4T · 12 GiB · 1×512 GB SATA SSD | etcd |
| soyo-2 | 10.0.0.21 | control-plane | N150 4C/4T · 12 GiB · 1×512 GB SATA SSD | etcd |
| soyo-3 | 10.0.0.22 | control-plane | N150 4C/4T · 12 GiB · 1×512 GB SATA SSD | etcd — **address by IP** (hostname flaky) |
| fringe-workstation | 10.0.0.23 | worker | i7-4770 4C/8T · 16 GiB · 256 GB SSD + **1 TB HDD** | write-heavy lands here |

soyo-1/2/3 each share their single 512 GB SSD across etcd + OS + all Longhorn replicas. `fringe-workstation` is the only node with a dedicated data disk (the 1 TB HDD) and the most RAM — put write-heavy workloads there via hard nodeAffinity `node-role.kubernetes.io/control-plane DoesNotExist`. Full specs + re-capture commands: `docs/techdocs/docs/general/talos-cluster.md` (Hardware).

## ⚠️ Apply always reboots — use the safe wrapper (drain→apply→wait→uncordon)
```bash
task talos:apply-node-safe IP=<ip> HOSTNAME=<name>    # never bare apply-node (no drain). soyo-3: IP=10.0.0.22
```

## Recipes (`.taskfiles/talos/Taskfile.yaml`)
`talos:generate-config` (after editing talconfig/patches/talenv) · `talos:apply-node-safe` · `talos:upgrade-node IP=<ip>` · `talos:upgrade-k8s` · `talos:reset` (DESTRUCTIVE wipe).
**Version bump:** edit `talenv.yaml` → `generate-config` → `upgrade-node` per node (one at a time) and/or `upgrade-k8s`.
