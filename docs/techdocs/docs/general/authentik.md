# Authentik

Authentik is the cluster identity provider in the `authentik` namespace.
Flux installs the official `goauthentik/authentik` Helm chart, CloudNativePG owns the PostgreSQL database, and Envoy Gateway exposes the UI at `https://authentik.${SECRET_DOMAIN}`.

## GitOps layout

- App Flux Kustomization: `kubernetes/apps/authentik/ks.yaml`
- Database Flux Kustomization: `kubernetes/apps/authentik/database/ks.yaml`
- HelmRelease: `kubernetes/apps/authentik/app/helmrelease.yaml`
- CNPG cluster: `kubernetes/apps/authentik/app/database/cluster.yaml`
- Grafana dashboard: `kubernetes/apps/observability/grafana/app/dashboards/security-authentik.yaml`
- Prometheus rules: `kubernetes/apps/observability/victoria-metrics/app/rules/prometheusrule-security-authentik.yaml`

## Blueprints

Custom blueprints are mounted as a ConfigMap (`authentik-blueprints`) and referenced in the HelmRelease via `blueprints.configMaps`. The ConfigMap is generated by a Kustomize `configMapGenerator` in `kubernetes/apps/authentik/app/kustomization.yaml`.

> Blueprints hardcode the **literal** cluster domain in their `context.domain` — the
> configMapGenerator output is not `${SECRET_DOMAIN}`-substituted.

### Dependency ordering

Authentik discovers blueprints in **alphabetical filename order** and applies each blueprint atomically. When one blueprint references objects from another (via `!Find`, `!KeyOf`, etc.), **declare the dependency explicitly** using the `authentik_blueprints.metaapplyblueprint` meta model. This ensures the prerequisite blueprint is fully applied before the dependent one runs — no race conditions, no manual recovery.

Add `metaapplyblueprint` entries at the **top** of the dependent blueprint's `entries` list:

```yaml
entries:
  - model: authentik_blueprints.metaapplyblueprint
    attrs:
      identifiers:
        name: Homelab - Core groups and users
      required: true
```

- `attrs.identifiers` matches the `metadata.name` of the prerequisite blueprint.
- `attrs.required: true` fails the entire blueprint if the prerequisite doesn't exist (safe default).
- Multiple `metaapplyblueprint` entries can declare multiple dependencies.

**This repo's dependency chain** (`kubernetes/apps/authentik/app/blueprints/`):

```text
00-core-groups-users.yaml   (no deps)
10-policies.yaml            (no deps, expressions are just strings)
20-flows-mfa-auth.yaml      (no deps, only internal !KeyOf)
30-oidc-grafana.yaml        → depends on all three above
33-oidc-backstage.yaml      → depends on all three above
34-oidc-forgejo.yaml        → depends on all three above
35-oidc-openbao.yaml        → depends on all three above
36-oidc-harbor.yaml         → depends on all three above
```

Every OIDC provider blueprint (`3x-*`) needs explicit `metaapplyblueprint` entries for the three
prerequisites; `10-` and `20-` are self-contained. (Former `31-oidc-n8n` and `32-oidc-searxng`
were removed — n8n's native OIDC is Enterprise-paywalled and SearXNG has none.)

## Adding a new OIDC application

**The step-by-step recipe (blueprint template, redirect-URI matching, ESO wiring, debugging) is
the `authentik-oidc` skill** — use it. Summary of the three moving parts:

1. **Blueprint** — `kubernetes/apps/authentik/app/blueprints/<nn>-oidc-<app>.yaml`, copied from an
   existing `3x-oidc-*.yaml`: an `oauth2provider` (authorization_code), an `application` bound via
   `!KeyOf`, and `policybinding` entries for `homelab-users-only` + `homelab-mfa-required`. The
   application `slug` determines the well-known URL:
   `https://authentik.${SECRET_DOMAIN}/application/o/<slug>/.well-known/openid-configuration`.
   `redirect_path` must match the app's OAuth callback exactly.
2. **Register** the filename in the `configMapGenerator` list in
   `kubernetes/apps/authentik/app/kustomization.yaml`.
3. **App-side config** — OIDC endpoints as env/config; client credentials as an
   **ExternalSecret** (OpenBao KV or a provisioner that reads them from the Authentik API — see
   `harbor-oidc-client.externalsecret.yaml` for the pattern). No SOPS secrets.

DNS prerequisite: the app pod must resolve `authentik.${SECRET_DOMAIN}` — failures show as
"no such host"; see the [split-DNS runbook](../runbooks/dns-split-dns.md).

## Retrieving OIDC credentials from Authentik

After blueprints are applied and processed (up to 10 min after Flux reconciliation), get credentials via the Authentik API:

```bash
# Get bootstrap token
TOKEN=$(mise exec -- kubectl get secret authentik-secret -n authentik -o jsonpath='{.data.AUTHENTIK_BOOTSTRAP_TOKEN}' | base64 -d)

# Query a provider
mise exec -- kubectl exec -n authentik deployment/authentik-server -- \
  curl -s -H "Authorization: Bearer $TOKEN" \
  'http://localhost:8000/api/v3/providers/oauth2/?name=<provider-name>' | jq '.results[0] | {client_id, client_secret}'
```

Or via the UI: **Authentik Admin → Applications → (app) → Provider**. Store the values in OpenBao
(`bao kv put secret/<app>/oidc …`) for the app's ExternalSecret to consume.

## Secrets

All Authentik secrets are **ESO-managed** (no SOPS):

- `authentik-secret.externalsecret.yaml` — `AUTHENTIK_SECRET_KEY` + bootstrap
  password-hash/token/email, sourced from OpenBao (`external-secrets` skill for changes).
- `harbor-oidc-client.externalsecret.yaml` — example of an app consuming Authentik-minted client
  credentials from OpenBao.
- The database password needs no managed secret: CNPG generates `authentik-db-app` and the
  HelmRelease injects its `username`/`password` keys.

## Operations

Check Flux and Helm:

```bash
mise exec -- flux get kustomizations -n authentik authentik-db authentik
mise exec -- flux get helmreleases -n authentik authentik
```

Check pods and database:

```bash
mise exec -- kubectl get pods -n authentik -l app.kubernetes.io/instance=authentik
mise exec -- kubectl get cluster -n authentik authentik-db
mise exec -- kubectl get secret -n authentik authentik-db-app
```

Dump Authentik's effective config:

```bash
mise exec -- kubectl exec -n authentik deploy/authentik-worker -- ak dump_config
```

The media PVC is `authentik-media`. If it approaches capacity, expand the PVC and let Longhorn grow the volume.
OIDC login failures: [Authentik OIDC login runbook](../runbooks/authentik-oidc-login.md).
